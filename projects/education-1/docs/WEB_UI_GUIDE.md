# 카프카 AI 웹 UI 사용 가이드

## 실행 방법

```bash
python main.py
```

실행 시 **http://localhost:8080** 에서 웹 UI가 열립니다.

---

## 화면 구성 및 사용 방법

### 1. 메인 화면 (홈)

**URL:** `http://localhost:8080/`

**보여지는 내용:**
- **제목:** 카프카 AI - 에빙하우스 망각 곡선 기반 복습 퀴즈
- **입력란:** URL 또는 텍스트를 입력하는 텍스트 필드
- **즉시 처리 버튼:** 입력한 URL/텍스트를 바로 처리하고 퀴즈 페이지로 이동
- **대기열에 추가 버튼:** 입력한 URL을 큐에 저장 (스케줄러가 나중에 처리)
- **대기 현황:** 현재 대기 중인 URL 개수 표시

**사용 방법:**
1. 입력란에 아래 중 하나를 입력
   - 뉴스 기사 URL (예: `https://n.news.naver.com/article/...`)
   - 유튜브 URL (예: `https://www.youtube.com/watch?v=...`)
   - 직접 입력할 텍스트
2. **즉시 처리** → 콘텐츠 분석 후 지식형이면 퀴즈 페이지로 이동  
3. **대기열에 추가** → 큐에 저장 (스케줄러가 매일 1개씩 처리)

---

### 2. 즉시 처리 플로우

**동작 순서:**
1. 사용자가 URL/텍스트 입력 후 **즉시 처리** 클릭
2. 서버에서 AI 파이프라인 실행 (분류 → 요약 → 퀴즈 생성 등)
3. **지식형** 콘텐츠 → `/quiz/{schedule_id}/1` 퀴즈 페이지로 **자동 이동**
4. **힐링형** 콘텐츠 → 메인 화면으로 돌아가며 "힐링형 콘텐츠" 안내 메시지 표시

**에러 시:**
- 유효하지 않은 URL, 유해 콘텐츠 등 → 메인 화면으로 돌아가며 빨간색 에러 메시지 표시

---

### 3. 퀴즈 화면

**URL 형식:** `http://localhost:8080/quiz/{schedule_id}/{notification_index}`

**보여지는 내용:**
- **헤더:** 카프카 복습 퀴즈, N차 복습, 페르소나 스타일
- **요약 박스:** 해당 콘텐츠 3줄 요약
- **문제:** 1개 문제 (4지선다 A/B/C/D)
- **제출하기 버튼:** 선택한 답 제출

**사용 방법:**
1. 4지선다 중 하나 선택
2. **제출하기** 클릭
3. 정답/오답 확인
4. 오답 시 → 다음날 같은 문제 재발송 예약

---

### 4. 대기열에 추가

**동작:**
- 입력한 URL을 DB 대기열에 저장
- 스케줄러(`python -m agent.scheduler.scheduler_service`)가 매일 1개씩 처리
- 처리된 콘텐츠는 스케줄에 저장되고, 에빙하우스 날짜에 알림 발송

---

## 요약

| 화면        | URL                         | 역할                             |
|-------------|-----------------------------|----------------------------------|
| 메인        | `/`                         | URL 입력, 즉시 처리, 대기열 추가 |
| 퀴즈        | `/quiz/{id}/{n}`            | 1문항 퀴즈 풀기                  |
| 퀴즈 제출   | `/quiz/{id}/{n}/submit` POST| 답안 제출 및 채점                |

## 참고

- `.env` 파일에 `UPSTAGE_API_KEY` 설정 필요
- 즉시 처리는 수십 초~수 분 걸릴 수 있음
- CLI 모드: `python main.py --url "URL"` 또는 `python main.py --text "텍스트"`
